splunk 之高级时间范围选项

最近学习 splunk 有遇到使用高级时间范围选项的场景，网上没找到太多相关学习资料，涉及到的也是一提而过，结合官方文档和实现需求的过程写个备忘录。

需求场景

写日报和周报需要从 splunk 上取数据，前同事留下的仪表板进去默认是最近 24 小时，每次进去要手动选择时间，所以开始摸索怎么能预设号时间。

开始在网上看了几篇相关介绍，大都一提而过没有很详细，使劲儿措辞了半天问 AI 给的结果也不满意，对于一个初学者确实有点捉急。

兜兜转转，果然还是得看官方文档。

官方文档说明

Use Advanced time range options
使用高级时间范围选项

Use the Advanced option to specify the earliest and latest search times. You can write the times in UNIX time or relative time notation, such as -3d@d. The UNIX time value you type is converted to local time.
使用“高级”选项指定最早和最晚的搜索时间。您可以用UNIX时间或相对时间表示法（如 -3d@d ）来编写时间。键入的UNIX时间值将转换为本地时间。

The UNIX time or relative time that you specify is displayed as a timestamp under the text field so that you can verify your entry.
您指定的UNIX时间或相对时间将在文本字段下显示为时间戳，以便您可以验证输入。

需求实现

虽然没有看很懂，但是已经能够满足我的使用需求了。

周报

周报的时间是七天前 0 点到今天早上 0 点

-7d@d``-0d@d

日报

日报的时间是昨天 15:30 到今天 15:30

-1d@d+15h+30m -0d@d+15h+30m

调整默认时间

调整后进仪表盘发现时间还是最近24小时，原来是还要调整默认时间。
